# Theming and Presets

MiniWiki uses **site-wide presets** for overall look and structure.

## Available presets

- `docs`
- `minimal`
- `portal`

## Change preset

Update `content/site.json`:

```json
{
  "wikiPreset": "minimal"
}
```

This changes the shell styling across header, sidebar, content area, and footer.

## Override preset on a single page

You can override the site preset in any page frontmatter:

```mdx
---
title: Example Page
wikiPreset: minimal
---
```

Supported values: `docs`, `minimal`, `portal`.

## Cozy magenta and other style variants (templates)

The `aura`, `forest`, and `midnight` variants are available as **page templates** now (not wiki presets).

Use one site-wide in `content/site.json`:

```json
{
  "defaultTemplate": "aura"
}
```

Override on a page with frontmatter as usual:

```mdx
---
title: Example Page
template: forest
---
```

## Edit-page link controls (frontmatter)

Per page, you can control the edit-page button:

```mdx
---
title: Internal Page
editPage: false
---
```

Optional custom edit URL override:

```mdx
---
title: Custom Edit Target
editPageUrl: https://example.com/my-custom-edit-target
editPageText: Propose an edit
---
```

## Next/previous docs navigation (frontmatter)

MiniWiki automatically shows Previous/Next page buttons for pages that are present in `content/sidebar.json` (using sidebar order).

You can disable either button per page:

```mdx
---
title: Example Page
disablePrevPage: true
disableNextPage: false
---
```

- `disablePrevPage: true` hides only the Previous button.
- `disableNextPage: true` hides only the Next button.
- If a page is first/last in sidebar order, only the available button is shown.

## Global edit-page config

Configure default behavior in `content/site.json`:

```json
{
  "editPage": {
    "enabled": true,
    "provider": "github",
    "repository": "OmgRod/MiniWiki",
    "branch": "main",
    "contentPath": "content",
    "text": "Edit this page"
  }
}
```

Use presets to brand one wiki differently from another without changing core code.
</Callout>
