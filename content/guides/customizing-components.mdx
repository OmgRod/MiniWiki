# Customizing Components

## Where components live

- All reusable docs components are in `components/`
- MDX registration is in `components/mdx-components.js`

## Add a new custom component

1. Create `components/MyComponent.js`
2. Export a React component
3. Register it in `components/mdx-components.js`
4. Use it in MDX pages

## Example registration

```js
import MyComponent from './MyComponent';

const mdxComponents = {
  ...existing,
  MyComponent,
};
```

## Style changes

- Prefer Tailwind utility updates inside the component file
- Use `styles/globals.css` for global markdown behavior only

## Safe customization checklist

- Keep component APIs simple
- Avoid hardcoded content in components
- Document props in the guide where it is used

<Callout icon="tool">
If a style change affects all docs content, place it in `styles/globals.css`.
</Callout>
